# AI-Лето: 7 проектов для школьника

Статический MVP-лендинг цифрового комплекта для детей 10–16 лет, взрослых наставников, семей, лагерей, санаториев, детских центров и летних площадок.

Комплект состоит из электронной рабочей тетради ребенка и методички наставника. Сайт поддерживает B2B2C-модель: одна организация получает один промокод, родители используют его при покупке, а учет продаж и партнерской комиссии ведется по коду.

Backend, база данных, авторизация и внешняя аналитика не используются.

## Конфигурация

Все изменяемые параметры находятся в `config.js`:

```js
const SITE_CONFIG = {
  productName: "AI-Лето: 7 проектов для школьника",
  productPrice: "999 ₽",
  promoDiscount: "50%",
  promoPrice: "499,50 ₽",
  partnerCommission: "20%",
  paymentUrl: "PASTE_PAYMENT_URL_HERE",
  organizationFormUrl: "PASTE_ORGANIZATION_FORM_URL_HERE",
  contactEmail: "PASTE_EMAIL_HERE"
};
```

### Ссылка на оплату

Замените `PASTE_PAYMENT_URL_HERE` в `paymentUrl` на публичную ссылку страницы оплаты. Все кнопки покупки используют это значение.

Если ссылка не задана, сайт показывает сообщение и не пытается открыть несуществующую страницу.

### Форма для организаций

Замените `PASTE_ORGANIZATION_FORM_URL_HERE` в `organizationFormUrl` на ссылку внешней формы — например, формы заявки для лагерей и детских центров.

Если ссылка не задана, CTA показывает fallback-сообщение с адресом из `contactEmail`.

### Цена, скидка и комиссия

- `productPrice` — базовая цена `999 ₽`;
- `promoDiscount` — заявленная скидка `50%`;
- `promoPrice` — цена после скидки `499,50 ₽`;
- `partnerCommission` — комиссия организации `20%`.

После изменения значений проверьте настройки платежной страницы: конфиг меняет отображение сайта, но не цену в платежном сервисе.

## Как работает промокод на статическом сайте

1. Посетитель вводит код организации.
2. Сайт нормализует код, показывает предварительную цену и сохраняет код в `localStorage` под ключом `aiSummerPromoCode`.
3. При покупке код добавляется к `paymentUrl` как параметр `promo`.
4. Существующие параметры ссылки оплаты сохраняются.
5. UTM-метки из адреса лендинга также переносятся в ссылку оплаты.

Пример:

```text
Лендинг: https://site.example/?utm_source=camp
Код: CAMP-SOCHI
Оплата: https://pay.example/product?utm_source=camp&promo=CAMP-SOCHI
```

Сайт не утверждает, что код подтвержден: действительность промокода и реальная скидка должны проверяться платежной страницей или вручную.

## Аналитика

Последние 100 событий сохраняются только в браузере в ключе `aiProjectsAnalytics` и выводятся в Console с префиксом `[AI-Лето]`.

Основные события:

- `promo_entered`;
- `promo_applied`;
- `buy_with_promo_click`;
- `buy_without_promo_click`;
- `organization_cta_click`;
- `hero_buy_click`;
- `pricing_buy_click`;
- `final_buy_click`;
- `faq_open`;
- `payment_url_missing`.

Проверка: откройте DevTools → Application → Local Storage и DevTools → Console.

## Ограничения без backend

В текущем MVP нельзя:

- реально валидировать промокоды на лендинге;
- автоматически считать и выплачивать партнерскую комиссию;
- автоматически менять лимит промокода с 10 до 20 или 30 применений;
- автоматически выдавать уникальные PDF каждому покупателю;
- защищать публично размещенный PDF от копирования.

На старте учет промокодов, подтвержденных оплат, лимитов и партнерских выплат ведется вручную через платежи и рабочую таблицу. Реальная валидация скидки должна происходить на стороне платежного сервиса.

## Материалы продукта

Не размещайте полный платный комплект в публичном репозитории. Загружайте электронную тетрадь и методичку наставника в платежный сервис или закрытое хранилище. В репозитории можно хранить только публичный демо-фрагмент.

Черновик продукта: `product/7-ai-projects-for-school-students.md`.

## Публикация на GitHub Pages

1. Отправьте ветку `main` в GitHub.
2. Откройте **Settings → Pages**.
3. Выберите **Deploy from a branch**.
4. Выберите `main` и `/(root)`.
5. Сохраните и дождитесь публикации.
6. Проверьте покупку без промокода, с промокодом, UTM-метки и CTA для организаций.

Сборка не требуется. Файл `.nojekyll` отключает обработку Jekyll.

## Перед запуском

- указать `paymentUrl`, `organizationFormUrl` и `contactEmail`;
- создать один промокод на каждую организацию в платежном сервисе;
- настроить скидку, лимит применений и учет подтвержденных продаж;
- загрузить два файла комплекта в закрытый сценарий выдачи;
- проверить юридические условия партнерской комиссии и лицензирования;
- протестировать сайт на телефоне и компьютере;
- не добавлять в репозиторий токены, пароли и персональные данные.
