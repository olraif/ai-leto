# 7 AI-проектов для школьника на лето

Статический MVP-лендинг одного цифрового продукта: PDF / онлайн-тетради с семью творческими AI-проектами для школьников 10–16 лет. Сайт помогает проверить спрос среди родителей и переводит покупателя на страницу товара в Plati.

Проект не использует backend, базу данных, авторизацию, сборщик или внешнюю аналитику. Для публикации достаточно содержимого репозитория.

## Локальный запуск

Откройте PowerShell в папке проекта и запустите любой простой HTTP-сервер.

Вариант с Python:

```powershell
python -m http.server 8000
```

Если команда `python` не найдена, попробуйте:

```powershell
py -m http.server 8000
```

Затем откройте `http://localhost:8000`. Сайт можно открыть и двойным кликом по `index.html`, но HTTP-сервер точнее повторяет работу GitHub Pages.

## Настройка продукта

Все изменяемые параметры находятся в [`config.js`](config.js):

```js
const SITE_CONFIG = {
  paymentUrl: "PASTE_PLATI_PRODUCT_URL_HERE",
  productPrice: "490 ₽",
  productName: "7 AI-проектов для школьника на лето",
  contactEmail: "PASTE_EMAIL_HERE"
};
```

### Ссылка на оплату

Замените `PASTE_PLATI_PRODUCT_URL_HERE` на полную ссылку товара Plati, например `https://example.com/product`. Все три кнопки покупки используют это значение. Query-параметры текущего адреса, включая UTM-метки, добавляются к ссылке оплаты, если в ней еще нет параметров с такими именами.

Если ссылка оставлена пустой, равна `#` или начинается с `PASTE_`, сайт не падает: показывает уведомление, записывает событие `payment_url_missing` и переводит посетителя к составу набора.

### Email

Замените `PASTE_EMAIL_HERE` в `config.js` на контактный адрес. После этого ссылка в подвале автоматически станет активной `mailto:`-ссылкой.

### Цена и название

Измените `productPrice` или `productName` в `config.js`. Цена обновится во всех CTA, карточке цены и доступных подписях кнопок, название — в карточке цены. При изменении цены также проверьте текст title/description и документацию Plati.

## Проверка CTA и локальной аналитики

1. Откройте DevTools браузера (`F12`).
2. Перейдите на вкладку Console.
3. Нажмите CTA или откройте пункт FAQ.
4. В консоли появится запись с префиксом `[Лето с AI]`.
5. Вкладка Application → Local Storage → адрес сайта содержит ключ `aiProjectsAnalytics`.

События: `hero_buy_click`, `hero_details_click`, `pricing_buy_click`, `final_buy_click`, `faq_open`, `payment_url_missing`. Хранятся последние 100 событий. Эти данные остаются только в браузере посетителя и никуда не отправляются.

Для проверки UTM откройте:

```text
http://localhost:8000/?utm_source=test&utm_campaign=summer
```

Укажите тестовый `paymentUrl`, нажмите кнопку и проверьте, что параметры появились в адресе перехода.

## Как добавить реальный PDF

Основной сценарий — загрузить PDF в карточку товара Plati и выдавать его после оплаты. Это предпочтительнее публичного хранения продаваемого файла в репозитории.

Если нужна публичная демо-версия, создайте папку `assets/files/`, положите туда файл с фрагментом продукта (например, `preview.pdf`) и добавьте ссылку на него в `index.html`. Не размещайте полный платный продукт в публичном репозитории, если доступ должен выдаваться только покупателям.

Черновик содержимого продукта находится в [`product/7-ai-projects-for-school-students.md`](product/7-ai-projects-for-school-students.md).

## Публикация на GitHub Pages

1. Создайте пустой публичный репозиторий GitHub.
2. Свяжите локальный проект с репозиторием и отправьте ветку `main`:

   ```powershell
   git remote add origin https://github.com/USERNAME/REPOSITORY.git
   git push -u origin main
   ```

3. В GitHub откройте **Settings → Pages**.
4. В разделе **Build and deployment** выберите **Deploy from a branch**.
5. Выберите ветку `main`, папку `/(root)` и нажмите **Save**.
6. Дождитесь публикации и откройте адрес, показанный в разделе Pages.
7. Проверьте меню, FAQ, все CTA и мобильную версию.

Файл `.nojekyll` отключает обработку Jekyll. Сборка не требуется.

## Подключение домена

Не переименовывайте `CNAME.example`, пока домен не выбран. После покупки домена создайте файл `CNAME` с одним доменным именем без `https://` и настройте DNS по инструкции [`docs/domain-setup.md`](docs/domain-setup.md).

## Настройка и обновление Plati

Полная инструкция и готовое описание товара находятся в [`docs/plati-setup.md`](docs/plati-setup.md). После создания или изменения товара скопируйте актуальную публичную ссылку и обновите только `SITE_CONFIG.paymentUrl` в `config.js`, затем проверьте три кнопки покупки.

## Перед запуском

- вставить рабочую ссылку Plati и контактный email;
- загрузить готовый PDF в Plati;
- проверить цену и условия получения материала;
- протестировать CTA с UTM-метками;
- проверить сайт на телефоне;
- при необходимости подключить домен и включить HTTPS;
- не добавлять в репозиторий токены, пароли, личные данные или полный закрытый PDF.
