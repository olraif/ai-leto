# Подключение домена к GitHub Pages

## Подготовка

1. Купите домен у выбранного регистратора.
2. Убедитесь, что сайт уже опубликован на стандартном адресе GitHub Pages и открывается по HTTPS.
3. В репозитории откройте **Settings → Pages** и найдите поле **Custom domain**.
4. Введите выбранный домен, например `www.example.ru` или `example.ru`, и сохраните.

## DNS для www-поддомена

У регистратора создайте запись:

- тип: `CNAME`;
- имя/хост: `www`;
- значение: адрес GitHub Pages вида `USERNAME.github.io` (без `https://` и пути).

## DNS для корневого домена

Для адреса без `www` используйте `A`, `AAAA` или `ALIAS/ANAME`-записи, которые на момент настройки рекомендуют GitHub Pages и ваш регистратор. Актуальные IP-адреса проверяйте в официальной документации GitHub, а не копируйте из старых инструкций.

## Файл CNAME

1. Скопируйте `CNAME.example` в новый файл с именем `CNAME`.
2. Оставьте в нем одну строку с доменом без протокола и пути:

   ```text
   www.example.ru
   ```

3. Закоммитьте и отправьте файл в ветку публикации.

Не создавайте настоящий `CNAME` до выбора домена: неверное значение может нарушить открытие сайта.

## Завершение

1. Дождитесь обновления DNS. Обычно изменения видны не сразу и могут распространяться до 24–48 часов.
2. Вернитесь в **Settings → Pages**.
3. Когда проверка домена завершится, включите **Enforce HTTPS**.
4. Откройте сайт по домену на компьютере и телефоне.
5. Проверьте сертификат HTTPS, favicon, Open Graph-превью, навигацию и переход на Plati.
6. Если используется и корневой домен, и `www`, настройте перенаправление одного варианта на основной у регистратора или хостинг-провайдера.

Важно: DNS-рекомендации и интерфейс GitHub могут меняться. При настройке сверяйтесь с актуальной официальной документацией GitHub Pages.
